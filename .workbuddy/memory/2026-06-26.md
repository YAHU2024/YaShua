# 2026-06-26 工作日志

## CircularProgress 环形进度盘不更新 BUG 修复（第三轮 — 真正根因）

### 问题
用户反馈：答题完成目标后返回首页，环形进度条和"已完成"数字均无变化。第二轮修复（onShow redraw）未解决问题。

### 根因分析（通过日志定位）
日志文件 `127.0.0.1-1782403075957.log` 中反复出现 6 次：
```
TypeError: requestAnimationFrame is not a function
    at CircularProgress.js
```

**微信小程序中没有全局 `requestAnimationFrame` / `cancelAnimationFrame`**。它们是 Canvas 2D 节点的方法：`canvasNode.requestAnimationFrame(cb)` / `canvasNode.cancelAnimationFrame(id)`。

代码中直接调用了全局 `requestAnimationFrame(frame)`，导致：
1. `initCanvas` → `drawRing` → `frame()` → `requestAnimationFrame(frame)` → 报错 → Canvas 完全空白
2. `watch` 回调 → `drawRing` + `animateValue` → 都调用全局 RAF → 都报错
3. `onShow` → `redraw()` → 仍然调用全局 RAF → 仍然报错

第二轮修复（onShow redraw）方向正确但无法生效，因为 redraw 内部同样调用了不存在的全局 RAF。

### 修复方案
在 `src/components/CircularProgress.vue` 中新增兼容封装函数：
- `raf(callback)`：优先 `canvasNode.requestAnimationFrame`，降级 `setTimeout(cb, 1000/60)`
- `caf(id)`：优先 `canvasNode.cancelAnimationFrame`，降级 `clearTimeout(id)`

替换全部 4 处调用：
1. drawRing 中 `cancelAnimationFrame(animFrameId)` → `caf(animFrameId)`
2. drawRing 中 `requestAnimationFrame(frame)` → `raf(frame)`
3. animateValue 中 `cancelAnimationFrame(valueAnimFrameId)` → `caf(valueAnimFrameId)`
4. animateValue 中 `requestAnimationFrame(frame)` → `raf(frame)`
