## 2025-06-25

### 修复首页今日学习-已完成/正确数量题数 BUG

**根因**（3 层问题叠加）：
1. 首页 stats 只在 `onMounted` 加载，刷题返回时 `onMounted` 不触发 → 数据停滞
2. `saveResults()` 写入云/本地后未同步更新 statsStore 响应式值
3. `todayQuestions`/`todayCorrect` 永远累加不归零，"今日"变成"累计"

**修改文件**：
- `src/pages/index/index.vue`：添加 `onShow` 生命周期，每次页面显示时刷新 stats 和错题数据
- `src/stores/quiz.ts`：新增 `syncStatsStore()` 在 saveResults 后同步更新 statsStore；`updateStats`/`saveResultsLocal` 增加跨日检测（新的一天用 SET 替代 INC）
- `src/stores/stats.ts`：`loadStats()` 增加每日重置逻辑，`updatedAt` 非今日则清零 `todayQuestions`/`todayCorrect`，并同步回云/本地存储

构建验证通过 (`npm run build:mp-weixin`)。

### 深度修复：syncStatsStore 覆盖今日数据的致命 BUG

**根因**：`syncStatsStore` 中 `_todayDate` 初始为空字符串 `''`，导致首次调用必定进入 SET（赋值）分支，用当前单次答题数**覆盖**已累积的今日数据。如已做5题+新做3题，应显示8但被覆写为3。

**修复**：
- `src/stores/quiz.ts`：删除 `_todayDate` 模块变量，`todayQuestions`/`todayCorrect` 统一 `+=` 累加（跨日重置由 loadStats + updateStats + saveResultsLocal 三层 SET 保障）
- `src/pages/index/index.vue`：`onShow` 中 openid 为空时先调 `doLogin` 确保就绪

构建验证通过。

### 第三轮修复：今日学习数据始终为0的根因修复

**根因**：两个机制互相打架——
- `syncStatsStore`（quiz.ts）用 `+=` 累加内存值 ✅
- `loadStats`（stats.ts）在 `onShow` 中用 `=` 从云端覆盖内存 — 云是0就覆盖成0 ❌

加上 `loadStats` 中 `isToday` 检测把 `todayQuestions:0` 写回云端（第56-63行），直接覆盖 `updateStats` 刚写入的云值。

**修复（5处）**：
1. `index.vue` onShow：移除 `loadStats`/`loadWrongQuestions` 调用，由 syncStatsStore 保证实时性
2. `quiz.ts` syncStatsStore：加 `_lastSyncDate` 跨日检测，新一天用 `=` 而非 `+=`
3. `quiz.ts` saveResults：云模式下始终调 `saveResultsLocal` 做本地容错备份
4. `stats.ts` loadStats：删除 isToday 中 `todayQuestions:0` 写回云/本地的危险代码
5. `quiz.ts` updateStats：返回 `boolean` 而非静默吞错误

构建验证通过。

### 第四轮修复：stats 只在交卷时更新，中途返回不更新

**根本根因**（日志证实 `[syncStatsStore]` 从未被调用）：
- 错题是逐题实时保存的（`confirmAnswer` → `wrongStore.addWrongQuestion`）
- 但 stats（今日学习计数）只在 `submitQuiz`（点击"提交试卷"）→ `saveResults` → `syncStatsStore` 时才更新
- **用户答题中途返回，stats 永远不更新**

**修复方案**：将 stats 更新从"交卷时批量"改为"每题确认时实时"，与错题收集模式一致

**修改**：
1. `quiz.ts`：新增 `recordSingleAnswer(isCorrect)` 每题实时更新 statsStore + 本地 storage；新增 `saveResultsLocalForSingle` 单题本地持久化
2. `quiz/index.vue`：`confirmAnswer` 中调用 `quizStore.recordSingleAnswer(isCorrect)`
3. `quiz.ts`：从 `saveResults` 中移除 `syncStatsStore` 调用和 `saveResultsLocal` 本地统计写入（避免重复计数）
4. `quiz.ts`：`updateStats` 改为从 statsStore 取当前值直接 SET（不用 INC），参数简化为仅需 openid

数据流修正：
```
confirmAnswer → recordSingleAnswer → statsStore ++ + 本地 storage ++  ← 每题实时
     ↓
submitQuiz → saveResults → updateStats(openid) → 云 SET statsStore 当前值  ← 最终同步
```
中途返回首页 → statsStore 已有实时数据 → 直接显示 ✅

构建验证通过。
