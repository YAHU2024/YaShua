## 修复题库 Markdown 导入字数限制

- **问题**: 用户反馈 Markdown 导入方式有字数长度限制，只能导入 1 题
- **根因**: uni-app（微信小程序）`<textarea>` 默认 `maxlength` 为 140 字符（旧版基础库行为），大段 Markdown 内容被截断
- **修复**:
  1. `src/pages/library/index.vue` — Markdown/JSON 导入 textarea 添加 `maxlength="50000"`（约可容纳 200+ 道题）
  2. `src/pages/library/index.vue` — H5 文件导入大小限制从 2MB 放宽到 10MB
  3. `cloudfunctions/importQuestions/index.js` — 内容安全检测截取从 2000 字符扩大到 10000 字符


## 微信开发者工具导入 & 日志问题修复

- **问题**: 项目导入微信开发者工具后报错 `TypeError: Cannot read property '_s' of undefined`（useUserStore/useLibraryStore）
- **根因 1（致命）**: `src/main.ts` 中 `createApp()` 未返回 `Pinia` 命名空间，导致 uni-app 小程序环境下 store 无法注入
- **根因 2（中等）**: 基础库 3.15.2 不再自动注入云开发 SDK，日志显示 `cloud sdk injection skipped`
- **修复**:
  1. `src/main.ts` — `import * as Pinia from "pinia"` + return `{app, Pinia}`
  2. `src/App.vue` — `onLaunch` 中添加 `wx.cloud.init({env:'', traceUser:true})`
  3. 已重新 `npm run build:mp-weixin`，构建成功，日志已确认两个修复均已注入
- **导入步骤**: 微信开发者工具 → 导入项目 → 目录选 `dist/build/mp-weixin`

## 第二轮日志修复：云适配器误选 & 云 init 失败

- **问题**: Pinia 修复后，新日志显示 `cloud init error`、`cloud.callFunction:fail`、`CloudAdapter.get(libraries) 失败`、`创建题库失败 errCode: -401003`
- **根因**: 虽云 SDK 未注入（`cloud sdk injection skipped`），但 `storageAdapter.ts` 通过 `uni.cloud` 判断云可用性，而 `uni.cloud` 是 uni-app shim 始终存在 → 错误选择了 `CloudAdapter`，导致所有 DB/云函数调用失败
- **修复**:
  1. `src/adapters/storageAdapter.ts` — `createStorageAdapter()` 判断从 `uni.cloud` 改为 `wx.cloud` + 验证 `wx.cloud.database` 存在，若云 SDK 未注入则自动降级为 `LocalAdapter`
  2. `src/App.vue` — `wx.cloud.init()` 包裹 try-catch，失败时静默降级不再报错
  3. 重新 `npm run build:mp-weixin` 成功

## 第三轮修复：云适配器仍误选（IDE 桩代码问题）

- **问题**: 第二轮修复后仍然使用 CloudAdapter，日志仍报 `CloudAdapter.get(libraries) 失败` 和 `errCode: -401003`
- **根因**: 微信开发者工具在云 SDK 未注入时仍提供 `wx.cloud` 桩代码（含 `database` 方法），但 `wx.cloud.version` 为 `undefined`。仅检查 `wx.cloud.database` 是函数不够，必须同时检查 `wx.cloud.version` 确保 SDK 已真正加载
- **修复**: `storageAdapter.ts` 云可用性判断增加 `wx.cloud.version` 检查

## 第四轮修复：文件上传 chooseFile 报错

- **问题**: 日志显示 `TypeError: e.index.chooseFile is not a function`，在 `pages/library/index.js` 中大量重复
- **根因**: `src/pages/library/index.vue` 的 `chooseFile()` 函数使用 `uni.chooseFile()`，该 API 在微信小程序中不存在（仅 App 端可用）。条件编译 `#ifdef APP-PLUS || MP-WEIXIN` 将两者混用导致小程序也走入了不存在的 API
- **修复**: 将 `APP-PLUS` 和 `MP-WEIXIN` 的条件编译分开：
  - `APP-PLUS`：保持 `uni.chooseFile()`
  - `MP-WEIXIN`：改用 `wx.chooseMessageFile({ count: 1, type: 'file' })`，从微信聊天记录中选择 .txt/.md 文件
  - `H5`：保持不变
- 重新 `npm run build:mp-weixin` 成功，编译产物中 `chooseMessageFile` 已替换 `chooseFile`
