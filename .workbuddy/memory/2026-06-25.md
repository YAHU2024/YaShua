## 错题本两级导航改造

### 需求
将错题本从"单页展开所有题库错题"改为"两级导航"：
- 第一级：显示各题库名称 + 错题数量（作为菜单）
- 第二级：点击题库后，展示该题库下的错题列表

### 实施内容

1. **修改 `src/pages.json`**：注册 `pages/wrong/detail` 页面

2. **新建 `src/pages/wrong/detail.vue`**（错题列表二级页面）
   - 接收 `libraryId` 和 `libraryName` 参数
   - 调用 `wrongStore.loadWrongQuestions(openid, libraryId)` 加载该题库错题
   - 手动组装题目详情（参考 `wrong.ts` 中 `getWrongQuestionDetails` 的逻辑）
   - 每道错题右侧显示 `×` 删除按钮，点击后调用 `wrongStore.deleteWrongQuestion(id)`
   - 底部固定操作栏："清空"按钮（确认弹窗）+ "全部重做"按钮（跳转刷题页）
   - 点击错题跳转到该题重做

3. **修改 `src/pages/wrong/index.vue`**（错题本首页）
   - 移除每个 group 下的 `.wrong-list`（错题列表）
   - 移除 `.group-actions`（"全部重做"和"清空"按钮）
   - 将 `.group-header` 改为可点击的 `.group-card`，点击后导航到 `detail.vue`
   - 显示：题库名称、错题数量、右箭头 ›

### 踩坑记录
- `onShow` 必须从 `@dcloudio/uni-app` 导入，不能从 `vue` 导入（构建会报错）
- `__unknown__` 题库的 `libraryId` 跳转时不能改成空字符串，否则 `detail.vue` 会加载所有错题

### 构建
- `npm run build:mp-weixin` 构建成功
- 产物路径：`dist/build/mp-weixin/`

## UI 全面优化 — Design Token 系统 + 组件化改造

### 背景
项目 6 页面 + 3 组件全部使用硬编码样式，无统一设计系统，维护困难。

### 新建文件（16 个）

**Design Token 系统（8 个）**：
- `src/styles/tokens/_colors.scss` — 品牌色/功能色/中性色
- `src/styles/tokens/_typography.scss` — 8 级字号/4 级字重/3 级行高
- `src/styles/tokens/_spacing.scss` — 8 点网格系统 + 语义化间距
- `src/styles/tokens/_radius.scss` — 5 级圆角
- `src/styles/tokens/_shadow.scss` — 4 级阴影层级
- `src/styles/tokens/_animation.scss` — 5 种时长 + 4 种缓动
- `src/styles/tokens/_breakpoints.scss` — 4 个断点 + safe-area
- `src/styles/tokens/_index.scss` — 汇总入口（import 全部 7 个模块）
- `src/styles/global.scss` — 全局工具类 + 关键帧动画定义

**共享组件（5 个）**：
- `src/components/LoadingState.vue` — 三环 CSS 旋转加载
- `src/components/EmptyState.vue` — 空状态（图标 + 标题 + 操作按钮）
- `src/components/ErrorState.vue` — 错误状态（重试按钮 + loading 态）
- `src/components/SkeletonCard.vue` — shimmer 骨架屏
- `src/components/BaseButton.vue` — 4 尺寸 × 4 变体统一按钮

**工具模块（1 个）**：

### 构建坑与修复
1. uni-app Vite 解析器不认识 SCSS partial 命名约定：`@import '@/styles/tokens/index'` 无法解析 `_index.scss`
   → 修复：全局替换为 `@import '@/styles/tokens/_index.scss'`（涉及 14 个文件）
2. `darken()` 在 Dart Sass 中已弃用 → 替换为直接计算值（`#f0f1f2`、`#fafafa`）
