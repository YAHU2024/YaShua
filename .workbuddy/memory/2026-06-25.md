## Bug 修复：错题本题目内容显示"已删除或无法加载"（第四轮）

### 根因
`getWrongQuestionDetails` 逐条调用 `db.collection('questions').doc(id).get()` 加载题目文档，
该方法的权限验证路径与 `where().get()` 不同，导致所有 `doc().get()` 返回空数据（`data: []`），
全部命中占位分支。

### 修复
将逐条 `doc().get()` 改为批量 `where({ _id: db.command.in(ids) }).get()`：
- 1次批量查询替代 N 次逐条查询，性能大幅提升
- 使用与正常刷题流程相同的 `where().get()` 代码路径，避开权限问题
- 分批100条防微信云 `in` 查询上限

### B1 致命链路
```
答错 → addWrongQuestion 云写入失败 → catch 降级写本地 ✅
→ 返回首页 → loadWrongQuestions → 云查询成功但空 → localSet([]) 覆盖本地 ❌
→ 本地被清空 → "暂无错题"
```

### 修复内容

| 修复 | 文件 | 说明 |
|------|------|------|
| B1 | wrong.ts | 云查询全量时（!libraryId）若返回空，回退本地缓存而非覆盖 |
| B3 | wrong.ts | 题目 doc 不存在时用占位 Question 兜底，不静默废弃错题 |
| B2 | quiz.ts | saveResults 单条错题写入 catch 中降级写入本地 storage |

| B6 | wrong.ts | addWrongQuestion 云写入/更新成功后加 `[错题]` 前缀日志 |

## Bug 修复：上传小程序时 `pages/library/index.wxml` 报 `unexpected character \n`

### 根因
`src/pages/library/index.vue` 中 3 处 `<textarea>` 的静态 `placeholder` 使用了 `&#10;` HTML 实体表示换行。uni-app 编译 WXML 时将其解码为真实 `\n`，导致 `placeholder` 属性跨行，WXML 解析器在第 2 行遇到 `\n` 报错。

### 修复
- 将 3 处静态 `placeholder` 改为动态 `:placeholder` 绑定到 `<script setup>` 中定义的字符串常量。
- 字符串常量中保留 `\n`，使换行符存在于 JS 运行时数据中，而非 WXML 属性值内。
- 为每个 `<textarea>` 添加 `placeholder-style="white-space: pre-line;"`，尽量保留多行提示排版。
- 全项目扫描 `src/` 中 `&#10;` / `&#13;` 等换行实体，仅余新增注释中的引用，无其他风险点。
