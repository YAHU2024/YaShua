## CircularProgress 环形进度盘 4 个 BUG 修复

### 问题
首页"今日学习进度"的 CircularProgress 组件存在 4 个显示 BUG：
1. **[严重] 进度永远 0% 或 100%**：`:total="Math.max(todayQuestions, 1)"` 使 total 始终等于 value
2. **[严重] "今日正确率"显示的不是正确率**：底部显示 value/total×100 而非 todayCorrect/todayQuestions×100
3. **[严重] Canvas 环形被裁剪**：drawRing 用硬编码 300（rpx 数值）作为绘制坐标，但 ctx.scale(dpr,dpr) 后坐标系为 CSS 像素（300rpx≠300px），导致环形超出画布可见区域
4. **[中等] 数字动画闪烁**：animateValue 未取消上一个 requestAnimationFrame，连续触发时多个动画叠加

### 修复方案

#### 1. `src/stores/stats.ts` — 新增 dailyGoal
- 新增 `dailyGoal` ref（默认 20 题），从本地 storage `daily_goal` 键初始化
- 新增 `setDailyGoal(goal)` 方法，写入 storage + 更新 ref
- 在 return 中暴露 `dailyGoal` 和 `setDailyGoal`

#### 2. `src/components/CircularProgress.vue` — 修复 BUG 2/3/4 + Canvas ID
- **BUG 3**：新增 `canvasCssW`/`canvasCssH` 模块级变量，initCanvas 中保存实际 CSS 像素尺寸；drawRing 中用 `canvasCssW/canvasCssH` 替代硬编码 `CANVAS_SIZE=300`，`r = min(w,h)*0.36`、`lineWidth = min(w,h)*0.0467`（保持原 108/300 和 14/300 的比例）
- **BUG 2**：新增 `accuracy?: number` prop + `displayPercent` computed（有 accuracy 用 accuracy，否则回退 percentage）；percentage 加 `Math.min(...,100)` 上限保护
- **BUG 4**：新增 `valueAnimFrameId` 变量，animateValue 开头取消上一帧，startVal 改为 `displayValue.value`（从当前值过渡）
- **额外**：Canvas ID 从硬编码 `cp-canvas` 改为 `cp-canvas-${random}` 唯一化，避免多实例冲突

#### 3. `src/pages/index/index.vue` — 修复 BUG 1
- `:total` 从 `Math.max(statsStore.todayQuestions, 1)` → `statsStore.dailyGoal`
- 新增 `:accuracy="statsStore.getTodayAccuracy()"`

#### 4. `src/pages/settings/index.vue` — 新增每日目标设置

- 使用 `uni.showActionSheet` 提供 10/20/30/50/100 题选项

## CircularProgress 环形进度盘不更新 BUG 修复（第二轮）

### 问题
用户反馈：答题完成目标后返回首页，环形进度条和"已完成"数字均无变化。

### 根因分析（后更正：此分析不准确，真正根因见第三轮）
答题时首页处于后台，`watch` 回调中的 `drawRing()` 和 `animateValue()` 依赖 `requestAnimationFrame`，以为页面后台时不触发。实际根因是 `requestAnimationFrame` 在微信小程序中根本不存在（全局），见第三轮修复。

### 修复方案
1. **`src/components/CircularProgress.vue`**：通过 `defineExpose` 暴露 `redraw()` 方法，调用 `drawRing(percentage.value)` + `animateValue(props.value)`
2. **`src/pages/index/index.vue`**：
   - 新增 `circularProgressRef` 模板 ref
   - 在 `onShow` 中通过 `nextTick(() => circularProgressRef.value?.redraw())` 强制重绘
   - `onShow` 在页面从后台恢复时触发（如从答题页 `navigateBack` 返回），此时页面已可见，`requestAnimationFrame` 正常工作

### 关键经验
- ~~微信小程序 Canvas 2D + `requestAnimationFrame` 动画在页面后台时不执行~~（更正：真正原因是全局 RAF 不存在，见第三轮）
- 页面 `onShow` 是重绘 Canvas 组件的正确时机
- 使用 `defineExpose` + 模板 ref 是 Vue3 组件通信的干净方式

### 构建验证
✅ `npm run build:mp-weixin` 构建成功（Exit Code: 0）
