## 1. 变更目标

## 3. 新功能需求

### 3.1 离线模式（本地存储降级）

**优先级**：P0

**背景**：H5 端与 App 端无法使用腾讯云开发 SDK，现有数据层直接报错。

**用户故事**：
- As a H5/App 用户，I want 题库和答题记录保存在本地，so that 我无需微信账号也能正常刷题。
- As a 用户，I want 在有网络时自动同步到云端，so that 多设备数据保持一致（P2，本期仅做本地存储，云同步不在范围）。

**需求描述**：
- `must`：封装统一存储适配层（`StorageAdapter`），根据运行环境自动选择腾讯云数据库或 `localStorage`。
- `must`：题库列表、题目数据、答题记录、错题集均支持本地读写。
- `should`：本地数据与云端数据结构保持一致，为后续同步预留接口。

---

### 3.2 文件导入（.txt / .md）

**优先级**：P1

**用户故事**：
- As a 出题者，I want 在手机文件管理器中选择已有的 .txt 或 .md 文件，so that 不必手动粘贴大段文本。

**需求描述**：
- `must`：题库导入弹窗新增「选择文件」按钮，调用 `uni.chooseFile`（App）/ `<input type="file">`（H5）。
- `must`：支持 `.txt` 和 `.md` 文件格式，文件大小限制 ≤ 2 MB。
- `should`：选择文件后自动填充到文本输入框，用户可预览再确认导入。
- `could`：上传进度提示（文件较大时）。

---

### 3.3 JSON 格式题库导入

**优先级**：P1

**用户故事**：
- As a 开发者/高级用户，I want 用标准 JSON 格式批量导入题目，so that 可以通过脚本批量生成题库。

**JSON 格式约定**（需在 UI 中提供示例）：

```json
[
  {
    "type": "single",
    "question": "题干",
    "options": ["A. 选项一", "B. 选项二"],
    "answer": "A",
    "explanation": "解析（可选）"
  }
]
```

**需求描述**：
- `must`：`parser.ts` 新增 `parseJSON` 方法，识别输入内容是否为合法 JSON 数组并解析。
- `must`：导入弹窗支持格式切换（Markdown / JSON / 文件），默认 Markdown。
- `should`：JSON 格式错误时给出具体的错误行提示，而非通用报错。

---

### 3.4 学习统计详情页

**优先级**：P1

**用户故事**：
- As a 学习者，I want 查看过去 7 天/30 天的做题量与正确率趋势，so that 了解自己的学习进展。
- As a 学习者，I want 按题库查看各题库的掌握度，so that 有针对性地复习薄弱题库。

**需求描述**：
- `must`：新增「统计详情」页面（路由：`/pages/statistics/index`）。
- `must`：首页「今日统计」区域可点击，跳转到统计详情页。
- `must`：统计详情页包含：
  - 总做题数、总正确率（累计）
  - 近 7 天每日做题量折线图（可用 uCharts 或原生 Canvas）
  - 各题库正确率列表（题库名 + 正确率百分比 + 进度条）
- `should`：支持切换「近 7 天 / 近 30 天」时间范围。
- `could`：连续打卡天数展示。

---

### 3.5 刷题数量设置

**优先级**：P2

**用户故事**：
- As a 用户，I want 开始刷题前设置本次刷题数量（10 / 20 / 50 / 全部），so that 根据当前时间灵活安排学习量。

**需求描述**：
- `must`：刷题入口（首页「开始刷题」/ 题库列表进入）增加底部弹出的设置面板。
- `must`：提供 4 个选项：10 题 / 20 题 / 50 题 / 全部，默认「20 题」。
- `must`：若题库题目数量不足所选数量，自动使用全部题目并给出提示。
- `should`：记住用户上次的选择（localStorage 持久化）。

## 4. UI/UX 要点

### 4.1 存储适配层（无 UI，架构变更）
- 对用户无感知；H5/App 环境下顶部可显示「本地模式」角标，提示数据仅存本地。

### 4.2 题库导入弹窗改版

```
┌─────────────────────────────┐
│  导入题目                    │
│  ─────────────────────────  │
│  格式：[Markdown] [JSON] [文件] ← Tab 切换
│                             │
│  [Markdown/JSON 输入框]      │
│  或                         │
│  [选择文件按钮] filename.md  │
│                             │
│  [取消]          [确认导入]  │
└─────────────────────────────┘
```

- 格式切换为 Tab 样式（3 个选项），默认选中 Markdown。
- 「选择文件」按钮在「文件」Tab 显示。
- 所有分支（成功/失败）均需调用 `closeModal()`，修复 BUG-04。

### 4.3 统计详情页布局

### 4.4 刷题数量选择面板

```
┌─────────────────────────┐
│  本次刷题数量            │
│  ○ 10题  ● 20题         │
│  ○ 50题  ○ 全部         │

```

- 底部半屏弹出（`uni-popup` type="bottom"）。
- 点击「全部」时，展示当前题库共 N 题提示文字。

## 5. 本期不在范围内

| 事项 | 原因 |
|------|------|
| 云端与本地数据双向同步 | 涉及冲突合并策略，复杂度高，独立排期 |
| AI 出题（aiParse 扩展） | 现有云函数基础稳定后再迭代 |
| 社交/分享功能 | 用户规模未达阈值，延后评估 |
| 多语言（i18n） | 当前目标用户为中文用户 |
| 题目评论/讨论 | 需后端支持，延后排期 |
